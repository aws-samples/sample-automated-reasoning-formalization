When you are making a code change to the business logic, always ask the senior engineer for a review.

When you are making a code change that inolves the asynchronous event flow, ask the principal engineer to review the change.

Before making changes to the UI or to a user experience flow, ask the principal UX design to review your proposal and give feedback.

After each change, ensure tests pass.

Do not deviate from the agreed design without asking the user for confirmation. Even if it seems like a big refactoring, always stick to the specifications and take the right path, do not implement temporary fixes or band aids.
